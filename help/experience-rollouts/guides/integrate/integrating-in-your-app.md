# Integrar implantações de experiência no seu aplicativo {#integrate}

Esta seção fornece diretrizes de integração para todos os clientes de Implantações de Experiência, organizadas por tipo de aplicativo.

## Escolha seu caminho de integração {#choose}

Selecione o guia que corresponde ao tipo de aplicativo:

1. [Guia de inicialização](startup-guide.md) — Comece aqui para obter uma visão geral de alto nível de todas as etapas de integração
2. [Aplicativos de desktop](desktop-applications.md) — Integração direta da API REST para aplicativos de desktop
3. [Aplicativos móveis](mobile-applications.md) — Integração da API REST para aplicativos móveis
4. [Aplicativos Web](web-applications.md) — Integração da API REST para propriedades da Web
5. [Serviços da Web](web-services.md) — Integração do SDK do lado do servidor para serviços de back-end
6. [SDKs](sdks.md) — arquitetura do SDK, pré-requisitos e SDKs disponíveis
7. [Etapas de integração](integration-steps.md) — Instruções detalhadas de integração passo a passo

## Principais tipos de aplicativos {#overview}

| Tipo de aplicativo | Integração recomendada |
|---|---|
| **Serviço Web/back-end** | Java SDK ou Node.js SDK |
| **Aplicativo Web** | REST API — API de recurso V3 |
| **Aplicativo móvel** | REST API — API de recurso V3 |
| **Aplicativo de desktop** | API REST — API de recurso V2 |
