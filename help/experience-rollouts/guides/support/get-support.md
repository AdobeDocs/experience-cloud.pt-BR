# Obter suporte {#get-support}

Se você encontrar um problema com as implantações de experiência que não pode ser resolvido usando a documentação ou as ferramentas de solução de problemas integradas, a equipe de suporte das implantações de experiência estará disponível para ajudar.

## Recursos de autoatendimento {#self-service}

Antes de enviar uma solicitação de suporte, tente estas opções de autoatendimento:

* **Pesquisar esta documentação** — Use a função de pesquisa para localizar guias, explicações de conceito e conteúdo de instruções.
* **Usar a Bancada de Recursos** — Diagnostica problemas de avaliação de sinalizador de recursos para usuários específicos sem escrever código. Consulte [Solução de problemas](troubleshooting.md) para obter instruções.
* **Verificar estados de versão** — Confirme se o recurso está no estado correto para o público-alvo desejado. Consulte [Estados de versão](../feature-flags/release-states.md).
* **Verificar regras de público-alvo** — Revise os critérios de público-alvo e a configuração de porcentagem do recurso. Consulte [Atualizar regras de lançamento de público-alvo](../feature-flags/update-release-audience-rules.md).

## Enviar uma solicitação de suporte {#submit-request}

Se os recursos de autoatendimento não resolverem o problema, envie uma solicitação de suporte. Consulte [Contate o suporte](contact-support.md) para obter detalhes sobre como entrar em contato com a equipe de suporte de Implantações de Experiência.

Ao enviar uma solicitação, inclua o máximo de detalhes possível para ajudar a equipe a investigar e resolver o problema rapidamente. Consulte a lista de verificação de informações em [Contate o suporte](contact-support.md).
